# 拆解MySQL索引：原理、分类与B+树底层逻辑详解

> 来源：博客园 https://www.cnblogs.com/bxxr/p/19713949
> 收集时间：2026-04-24

索引是 MySQL 优化查询性能的核心手段，如同书籍的目录，能帮助数据库快速定位数据，避免全表扫描的低效操作。本文将从索引的基础概念出发，详解索引的分类、优缺点、创建方式，以及底层的 B + 树结构原理，帮你彻底理解索引的工作机制。

## 一、索引的核心概念

### 1.1 索引的本质

索引本质上是数据库的 "目录"，通过预先构建的目录信息（键值 + 数据地址），让数据库无需全表扫描（逐行逐字段匹配），就能快速找到目标数据。

无索引 vs 有索引：

- 无索引：查询数据时需全表扫描，数据量越大，查询越慢；

- 有索引：通过索引结构直接定位数据位置，查询效率呈数量级提升。

### 1.2 索引的适用场景

#### ✅ 建议加索引的场景

- 以查询（SELECT）为主的字段（如用户 ID、订单号、商品名称）；

- 经常作为 WHERE 条件、JOIN 关联、ORDER BY 排序的字段。

#### ❌ 不建议加索引的场景

- 以写操作（INSERT/UPDATE/DELETE）为主的字段：写入数据时，需同步更新索引的底层数据结构（如 B + 树），会显著降低写入性能；

- 数据量极小的表（如配置表，全表扫描比走索引更快）；

- 重复值极高的字段（如性别字段，索引筛选效果差）。

### 1.3 索引的优缺点

优点：显著提升查询速度、降低磁盘 I/O 次数（索引按字段排序）、避免服务器排序/临时表操作

缺点：增加存储成本（索引需额外磁盘空间）、降低写入性能（写入需同步更新索引）、索引可能失效（如查询条件与索引不匹配）

## 二、索引的分类与创建方式

MySQL 索引按字段数量可分为单列索引和多列索引，其中单列索引是最常用的类型。

### 2.1 单列索引

单列索引仅针对单个字段创建，包含主键索引、唯一索引、普通索引三种核心类型。

#### （1）主键索引（PRIMARY KEY）

- 核心特性：非空（NOT NULL）+ 唯一（UNIQUE），一张表只能有一个主键索引；

- 自动创建：定义主键时，MySQL 会自动创建主键索引；

- 手动创建：
```sql
ALTER TABLE 表名 ADD PRIMARY KEY (索引字段);
ALTER TABLE test_index ADD PRIMARY KEY (id);
```

#### （2）唯一索引（UNIQUE）

- 核心特性：字段值唯一，允许 NULL 值；

```sql
CREATE UNIQUE INDEX idx_phone ON test_index (phone);
```

#### （3）普通索引（INDEX）

- 核心特性：无唯一性、非空约束，仅用于提升查询速度，是最常用的索引类型；

```sql
CREATE INDEX idx_price ON test_index (price);
```

### 2.2 多列索引（复合索引）

- 最左匹配原则：查询条件必须包含索引的第一个字段，索引才会生效

```sql
CREATE INDEX idx_id_name ON test_index (id, name);
```

## 三、索引的底层实现：从二叉树到 B + 树

MySQL 索引的底层默认采用B + 树结构，这是经过多代树形结构优化后的最优方案。

### 3.1 二叉搜索树（基础）

- 结构特点：每个节点最多 2 个子节点，左子节点值小于父节点，右子节点值大于父节点

### 3.2 AVL 树（平衡二叉树）

- 结构特点：基于二叉搜索树优化，强制要求每个节点的左右子树高度差≤1
- 缺点：节点仅存储单个数据，数据量大时树层级仍过高，磁盘 I/O 次数多

### 3.3 BTree（B 树）

- 结构特点：多叉树（单个节点可存储多个键值），内部节点和叶子节点都存储数据
- 缺点：仅适合精确查询，不支持高效的范围查询

### 3.4 B + 树（MySQL 默认）

B + 树是 BTree 的优化版本，也是 MySQL 索引的核心底层结构：

- 内部节点仅存储键值，叶子节点存储完整数据
- 所有叶子节点通过链表连接，支持快速范围遍历
- 插入/删除操作主要集中在叶子节点，稳定性好

核心优势：
- 减少磁盘 I/O：内部节点只存键值，单个节点能存储更多索引项，树的层级更少（通常 3-4 层即可支撑千万级数据）
- 范围查询高效：叶子节点链表有序，范围查询只需遍历链表
- 查询稳定性高：所有数据查询最终都要到叶子节点

## 四、核心总结

- 索引核心价值：通过 "目录" 机制避免全表扫描，提升查询速度
- 索引类型关键：主键索引（非空唯一）、唯一索引（唯一可空）、普通索引（无约束）
- 底层结构选择：MySQL 索引默认采用 B + 树，相比二叉树、B 树，其层级少、范围查询高效、稳定性强
