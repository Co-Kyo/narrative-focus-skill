## 被高亮的概念

### 1. B+树结构演进路径（第三章，~45%篇幅）— ✅ 权重合理

这是全文的叙事重心，也是修正后最显著的改善。从二叉搜索树 → AVL树 → B树 → B+树，每个阶段都有独立小节，包含：
- 结构特点说明
- 查找效率分析（含时间复杂度）
- 局限性分析（为什么不够好）
- 过渡性结论（blockquote 形式，承上启下）

**权重评价**：文章标题包含"B+树底层逻辑详解"，目标读者的核心需求是理解"为什么 MySQL 选择 B+树"。将树结构演进路径作为叙事主线，篇幅占比最大，是完全正确的选择。每个树结构都有"局限 → 改进"的因果铺垫，逻辑链条完整。

### 2. 索引的本质与核心概念（第一章，~30%篇幅）— ✅ 权重合理

作为入门铺垫，解释"索引是什么"、"什么时候该用"、"优缺点是什么"，篇幅适中。没有过度展开，也没有遗漏关键信息。

### 3. 三种索引类型（第二章中的表格，~15%）— ✅ 权重合理

主键/唯一/普通三种索引被压缩为一张对比表格，仅保留核心区别（约束条件、特点、创建示例）。没有各自独立成节，没有重复的代码块。作为"配置层面的补充说明"，这个权重是恰当的。

## 被压缩的概念

以下概念在当前文章中篇幅较少，但经分析均属于文章主线之外的内容，压缩是合理的：

### 1. 多列索引/最左匹配原则

仅一段话 + 一条 SQL 代码。篇幅虽少，但最左匹配的深度讲解应由专门讨论复合索引使用的文章承担，而非塞进一篇以 B+树底层原理为主线的文章。当前的提及足以让读者知道这个概念的存在，后续可以沿着正确的问题链去其他资源深入学习。

### 2. 索引失效场景

优缺点中一句话带过。索引失效是另一个独立话题（使用指南 vs 原理入门），不属于本文的叙事范畴。读者在理解了 B+树原理后，自然会追问"什么情况下索引不生效"——这个疑问可以由其他文章来回答。

### 3. B+树与 InnoDB 的具体关联

聚簇索引 vs 二级索引、回表查询等概念未涉及。这些是 B+树原理在 MySQL 中的具体应用层知识，超出了本文"原理入门"的定位。叙事重心正确的情况下，读者理解了 B+树结构后，会自然产生对这些具体实现的好奇——这就是正确的疑问链。

**小结**：以上三项均为"内容完备性"层面的缺失，而非"叙事逻辑"层面的问题。在叙事重心正确的前提下，这些内容的缺失不影响读者形成正向的学习路径——它们是读者在正确道路上会自然遇到的下一个问题，可以由其他资源来承接。
