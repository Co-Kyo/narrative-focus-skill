# 叙述重心评审报告：深入理解 Git - Git 底层对象

## 一、文章结构概述

文章围绕 Git 的四种底层对象（blob、tree、commit、tag）展开，辅以压缩存储和"为什么删除大文件要重写历史"两个补充话题。全文约 800 字，属于短文。

## 二、篇幅分配识别

| 概念 | 篇幅占比（估） | 优先级 |
|------|------------|--------|
| blob object | ~15% | 高（首先介绍） |
| tree object | ~5% | 中（一句话带过） |
| commit object | ~15% | 高 |
| 分支 = commit 引用 + 案例 | ~20% | 高（配有实战命令） |

**最大篇幅获得者**：分支引用案例 + 删除大文件重写历史，两者合计约 40%。

## 三、重心是否合理？

### ✅ 合理之处

1. **"分支是 commit 引用"的案例**：这个理解确实是 Git 底层模型的关键认知跃迁，用 `git update-ref` 对比 `git reset` 来具象化，是好设计。
2. **删除大文件重写历史的解释**：这是开发者实际会遇到的问题，且解释紧扣对象模型（tree 指向 blob、commit 不可变），把理论和实践串联起来了。

### ⚠️ 重心偏移之处

1. **tree object 被严重压缩**：只有一句话"相当于文件系统中的文件夹"。但 tree object 是连接 blob 和 commit 的关键纽带——它决定了 Git 如何表示目录结构、如何做 diff、如何做 merge。对于"深入理解 Git 底层对象"这个标题，tree object 至少应该和 blob 同等篇幅，解释其内部结构（条目列表：mode + name + hash）以及它如何递归嵌套。

2. **tag object 的权重偏高**：花了较多篇幅区分"标签对象不是引用"以及"一切皆 commit 对标签不一定正确"。这个辨析有价值，但对于目标读者（想理解 Git 底层机制的开发者），tag 的优先级应该低于 tree 和 commit 的内部结构细节。

3. **对象存储机制过于简略**：文章只用一段话概括"松散对象 → gc 压缩 → 增量存储"，但没有解释 packfile 的结构、delta 压缩的原理、`.git/objects/pack` 的存在。这些是理解 Git 存储效率和性能的关键，对于"深入理解"级别的文章来说是必要的。

4. **SHA-1 哈希与对象寻址几乎没提**：Git 的核心是 content-addressable storage——对象通过其内容的 SHA-1 哈希来寻址。文章开头提了一句"hash 值和校验值的组合"，但没有展开。这应该是全篇的理论基石。

## 四、被省略但应该是核心的内容

| 省略内容 | 重要性 | 说明 |
|---------|--------|------|
| tree object 内部结构 | 🔴 高 | 条目列表（mode, name, hash）、递归嵌套机制 |
| content-addressable storage 原理 | 🔴 高 | SHA-1/SHA-256 寻址是 Git 对象模型的根基 |
| packfile 与 delta 压缩 | 🟡 中高 | 理解 Git 为何高效、为何有 gc 概念 |
| 对象如何在 `.git/objects` 中物理存储 | 🟡 中 | loose object 的 zlib 压缩、路径生成规则（前2位做目录） |
| merge commit 的特殊性（多父提交） | 🟡 中 | 文章只提"父提交ID"，未区分单父/多父 |

## 五、整体评价

**叙述重心与读者需求的匹配度：60%（基本匹配但有明显偏差）**

文章抓住了 Git 对象模型的"四种对象"框架，并用分支引用和删文件重写历史两个实际场景来巩固理解，这个思路是对的。但问题在于：

- **tree object 被一笔带过**，而它是理解 Git diff、merge、文件系统映射的核心——这就像讲数据库只讲了行和表的概念，但跳过了索引结构。
- **"为什么删大文件要重写历史"占据了收尾的大量篇幅**，虽然实用，但它更像是一个推论/推论应用，不应该比 tree object 的内部结构获得更高权重。
- **对象的物理存储和寻址机制几乎缺席**，这让"深入理解"停留在概念层面，没有触及实现层面。

一句话总结：文章在"概念关系"层面讲得不错（四种对象的关系、分支=引用），但在"单个对象的内部结构"和"物理存储机制"这两个维度上投入不足，导致叙述重心偏应用推论而轻底层细节，与标题"深入理解 Git 底层对象"的承诺存在落差。
