```

- [❌ 错位-被高光] 事件委托：文中作为"三大机制之一"独立成章，实际是传输性（只是信号获取手段）
- [❌ 错位-标题内容错配] "事件委托的运行时流程"：标题指向委托，内容实际是 Fiber 遍历分发
- [❌ 错位-被掩盖] Fiber 树遍历：实际是架构性核心，但被压缩在分发流程的几行步骤里
- [❌ 错位-被掩盖] stopPropagation() 只在 Fiber 层生效：如此关键的行为差异，只在对比表一行和落点一句中出现
- [❌ 错位-篇幅过高] React 17 挂载点变化：配置性细节，占据机制二整章
```

## 关键错位点原文摘录

### 错位点 1：事件委托被当作核心机制

```markdown
#### 机制二：事件委托——React 17 前后的挂载点变化

（整章讲解 document → #root 的变化、微前端共存、迁移影响……）
```

替换实验：如果把事件委托换成逐元素绑定，用户写的 `onClick` 行为完全不变 → **传输性**，不应独立成章。

### 错位点 2：Fiber 遍历被压缩在流程步骤里

```markdown
#### 机制三：事件委托的运行时流程

阶段二：分发阶段（Event 触发时）

用户点击按钮
  → 原生 click 事件冒泡到 root container
  → React 的统一监听器被触发
  → React 从原生 event.target 向上遍历 DOM 树    ← 这一行就是 Fiber 遍历的核心
  → 对每个 DOM 节点，查找对应的 Fiber 节点         ← 这也是
  → 收集该事件类型的所有匹配处理器（capture → bubble 顺序）  ← 这也是
  → ...
```

替换实验：如果把 Fiber 遍历换成 DOM 树遍历，Fragment/Provider 上的 handler 会丢失 → **架构性**，应该独立展开。

### 错位点 3：stopPropagation 差异只在表格一行

```markdown
| `stopPropagation()` | **只阻止 React 内部冒泡**，不影响原生事件监听器 | 阻止原生冒泡 |
```

替换实验：如果这个行为变了，用户代码会出 bug → **架构性**，应该有独立的代码示例展开说明。
